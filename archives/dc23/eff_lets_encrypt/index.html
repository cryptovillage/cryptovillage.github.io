Sometimes need to create one of these... 
[req]
distinguished_name = req_distinguished_name
req_extensions = v3_req
default_md = sha256

[req_distinguished_name]
countryName = Country Name (2 letter code)
countryName_default = US
stateOrProvinceName = State or Province Name (full name)
stateOrProvinceName_default = California
localityName = Locality Name (eg, city)
localityName_default = San Francisco
organizationalUnitName  = Organizational Unit Name (eg, section)
organizationalUnitName_default  = Electronic Frontier Foundation
commonName = Common Name (ie hostname or username)
commonName_default = example.eff.org
commonName_max  = 64

[ v3_req ]
# Extensions to add to a certificate request
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
subjectAltName = @alt_names

[alt_names]
DNS.1 = example.eff.org
DNS.2 = www.example.eff.org
DNS.3 = example.com
DNS.4 = www.example.com

Then you need to...

  1. Copy/paste the CSR into a text box

  3. Download the certificate, once issued
  4. Download any intermediates

And then comes deployment...

  1. Enable SSL extensions for your web server
  2. Look up the appropriate config options
  3. Choose a ciphersuite (if you're savvy)
  4. Create a new virtualhost listening on 443
  5. Create redirects from unencrypted site
  6. Reload the web server

Enter Let's Encrypt

https://www.letsencrypt.org/

Internet Security Research Group

  • A software project that offers TLS certificates to anyone who can prove ownership of a domain (or subdomain)
  • For *free*
  • Zero deployment overhead

Let's Encrypt Development

  • Started as a collaboration between EFF, University of Michigan, Mozilla
  • Let's create a fully automated CA
  • Issuing certs for the lowest level of validation Domain Validation
  • Replace the manual CA UX with a piece of software

Install Let's Encrypt

# apt-get install lets-encrypt

Plays two roles:


  • Certificate Authority
  • Deployment Handler

Certificate Authority

  • Has the power to issue certificates that are publically trusted by the browser.


Certificate Authority

Certificate Authority

  • New protocol: Automatic Certificate Management Environment (ACME)
  • Prove control of domains through provisioning resources
  • Provisioning and validation happens completely in software
  • DNS, SimpleHTTP

DVSNI

  • Uses the Server Name Indicator (SNI) to prove Domain Validation (DV) ownership
  • Configures HTTPS server with a key the CA knows the requesting account holds

Deployment Handler

  • Reconfigures your webserver to add a new virtualhost
  • Deploys the certificate
  • Uses best-practice recommendations for ciphersuites
  • HSTS, Redirects
  • OCSP

Deployment Handler

  • Apache, NGINX
  • Plugin Architecture

Deployment Handler

  • Renewals happen automatically through ACME account manamgement
  • Revocations require manual intervention, are supported in ACME
